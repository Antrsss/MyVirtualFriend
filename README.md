# My Virtual Friend — инструкция по запуску

Одностраничное веб‑приложение на чистом JS: авторизация через Firebase Authentication, хранение чатов в Firestore и ответы от ИИ через **Node.js backend** (OpenRouter).

## 1. Структура проекта

- `index.html` — разметка приложения.
- `style.css` — зелёный минималистичный дизайн, закруглённые формы.
- `firebaseConfig.js` — конфигурация и инициализация Firebase.
- `app.js` — логика авторизации, работы с чатами и вызова ИИ через бэк.
- `server/` — Node.js backend для OpenRouter (ключ хранится на сервере).
- `README.md` — эта инструкция.

## 2. Настройка Firebase

1. **Создайте проект Firebase**
   - Зайдите на сайт Firebase (`https://console.firebase.google.com`).
   - Нажмите **Add project** / «Создать проект».
   - Введите имя проекта, подтвердите создание.

2. **Подключите веб‑приложение**
   - В консоли проекта нажмите значок `</>` (Web app / Веб‑приложение).
   - Задайте название, выберите «Также настроить Firebase Hosting» при желании.
   - После создания вам покажут блок кода `const firebaseConfig = { ... }`.

3. **Заполните `firebaseConfig.js`**
   - Откройте файл `firebaseConfig.js`.
   - Замените значения:
     ```js
     const firebaseConfig = {
       apiKey: "ВАШ_API_KEY",
       authDomain: "ВАШ_PROJECT_ID.firebaseapp.com",
       projectId: "ВАШ_PROJECT_ID",
       storageBucket: "ВАШ_PROJECT_ID.appspot.com",
       messagingSenderId: "ВАШ_SENDER_ID",
       appId: "ВАШ_APP_ID",
     };
     ```

4. **Включите Email/Password Auth**
   - В консоли Firebase откройте **Authentication → Sign-in method**.
   - Включите **Email/Password** (Пароль/Email).

5. **Создайте базу Firestore**
   - Перейдите в **Firestore Database**.
   - Нажмите **Create database**.
   - Выберите режим теста (или напишите свои правила) и регион.

6. **Простые правила безопасности (минимум для разработки)**
   В разделе **Rules** Firestore можно временно использовать:
   ```js
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /chats/{chatId} {
         allow read, write: if request.auth != null
           && request.auth.uid == resource.data.userId;
       }

       match /chats/{chatId}/messages/{messageId} {
         allow read, write: if request.auth != null;
       }
     }
   }
   ```
   Для продакшна правила нужно ужесточить.

## 3. Настройка ИИ через OpenRouter (Node.js backend)

1. Создайте аккаунт в OpenRouter и получите API key (переменная `OPENAI_API_KEY`).
2. В папке `server/` создайте файл `.env` по примеру `.env.example`:
   ```bash
   OPENAI_API_KEY=ваш_openrouter_key
   SITE_URL=http://localhost:8000
   PORT=8787
   ```
3. Установите зависимости и запустите сервер:
   ```bash
   cd server
   npm install
   npm run dev
   ```
4. Фронтенд шлёт запросы на `http://localhost:8787/api/chat`.
   - Роль и «тема разговора» формируют системный промпт (см. `server/src/prompts/roles.js`).
   - Контекст (последние сообщения) отправляется вместе с запросом.

> Важно: токен лучше не коммитить в публичный репозиторий.

## 4. Запуск приложения локально

1. Убедитесь, что все файлы находятся в одной папке (как в этом проекте).
2. **Откройте `index.html` напрямую в браузере** (двойной клик или через «Открыть с помощью»).
   - Либо поднимите простой статический сервер, например:
     - Python: `python -m http.server 8000`
     - Node (npm): `npx serve .`
3. В браузере откройте `http://localhost:8000` (если используете локальный сервер).
4. Убедитесь, что **бэк запущен** (см. раздел 3) — иначе ИИ не сможет отвечать.

## 5. Как работает интерфейс

- При входе вы видите экран авторизации:
  - Вкладки «Войти» и «Создать аккаунт».
  - Авторизация по email/паролю через Firebase Authentication.
- После входа:
  - Слева — список чатов (с ролями и временем последнего сообщения).
  - Сверху — выбор роли: друг, сестра, мама, папа, любимый человек и т.д.
  - Рядом — поле «О чём поговорим?» (тема/контекст, влияет на ответы ИИ).
  - По центру — активный чат.
  - Внизу — поле ввода сообщения.
- Сообщения хранятся в коллекции `chats` (документы пользователя) и подколлекции `messages` для каждого чата.
- При отправке сообщения:
  1. Сообщение пользователя сохраняется в Firestore.
  2. Вызывается Node.js backend, который обращается к OpenRouter.
  3. Ответ ИИ сохраняется как сообщение собеседника и отображается в чате.

## 6. Кастомизация

- Можно менять список ролей и их подписи в объекте `ROLE_LABELS` в `app.js`.
- Дизайн (оттенки зелёного, закругления, шрифты) редактируется в `style.css`.
- При желании можно:
  - добавить аватарки,
  - отображать имя пользователя,
  - показать индикатор «печатает…»,
  - заменить ИИ API на другой (OpenRouter, сторонний бэкенд и т.п.).

Если хочешь, могу отдельно расписать готовые правила безопасности для продакшна и/или помочь опубликовать сайт на Firebase Hosting.
